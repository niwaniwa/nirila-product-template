# リリースチェックリスト

> リリース前後に確認すべき事項の標準チェックリスト

## 使い方

- リリースごとにこのチェックリストをコピーして使用する
- プロジェクトの成熟に合わせて項目を追加・削除する
- 毎リリース後の振り返りで改善する

---

## Pre-release（リリース前）

### ドキュメント整合性

- [ ] 対象specが `status: accepted` であることを確認
- [ ] 関連ADRが `status: accepted` であることを確認
- [ ] `bash scripts/validate-docs.sh` がエラーなしで通過
- [ ] specの未解決事項がすべて解消済み
- [ ] 変更に伴うドキュメント更新が完了

### コード品質

- [ ] コードレビュー完了（AI + 人間）
- [ ] 全テスト通過（ユニット・結合）
- [ ] lint / format チェック通過
- [ ] 新規コードにテストが追加されている

### セキュリティ

- [ ] secretsがコードに含まれていない
- [ ] 依存パッケージの脆弱性チェック実施
- [ ] ユーザー入力のバリデーション確認

### デザイン

- [ ] デザイントークンとの一致確認
- [ ] UI/UXがspecの要件を満たしている

## 振り返り

リリース後に以下を記録し、次回のチェックリスト改善に活かす：

- 問題が発生した場合、チェックリストで防げたか
- 不要だった項目はあるか
- 追加すべき項目はあるか
