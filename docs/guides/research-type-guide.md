# research_type ガイド

> `docs/research/` のドキュメント作成時に `research_type` フィールドに設定する値のガイダンス

## research_type 一覧

| research_type | 対応フェーズ | 概要 | 主な成果物 |
|---|---|---|---|
| `competitive-analysis` | 要求整理 | 競合プロダクトの機能・価格・強み弱みの比較 | 比較表、差別化ポイント |
| `market-research` | 要求整理 | 市場規模・トレンド・ターゲットニーズの調査 | 市場分析レポート |
| `user-research` | 要求整理 | ユーザーインタビュー・行動分析・ペルソナ検証 | ペルソナ、ユーザージャーニー |
| `tech-evaluation` | 仕様策定・設計 | 技術・ライブラリ・フレームワークの比較選定 | 比較表、推奨技術 |
| `feasibility-study` | 要求整理・設計 | 実現妥当性の多面的検討 | 妥当性評価レポート |
| `poc-report` | 設計・実装 | 技術検証（PoC）の実施結果 | PoC結果、判断材料 |
| `gap-analysis` | 全フェーズ | 現状と目標のギャップ分析 | ギャップ一覧、対策案 |

---

## 各タイプの詳細

### competitive-analysis（競合分析）

Deep Researchを活用して競合を洗い出す。

**テンプレートに含めるべき項目:**
- 競合プロダクト一覧
- 機能比較表
- 価格・ビジネスモデルの比較
- 強み・弱みの分析
- 自プロダクトの差別化ポイント

### market-research（市場調査）

**テンプレートに含めるべき項目:**
- 市場規模（TAM/SAM/SOM）
- 成長率・トレンド
- ターゲットセグメント
- 参入障壁

### tech-evaluation（技術評価）

Deep Researchで候補を洗い出し、比較表で選定する。結果はADRに接続。

**テンプレートに含めるべき項目:**
- 評価対象の技術リスト
- 評価基準（パフォーマンス、学習コスト、コミュニティ、ライセンス等）
- 比較表
- 推奨と根拠
- → 関連ADRへのリンク

### feasibility-study（実現妥当性調査）

以下の5つの観点で妥当性を評価する：

| 観点 | 評価内容 |
|---|---|
| **Technical** | 技術的に実現可能か（スキル・ツール・インフラ） |
| **Economic** | コスト対効果（開発費・運用費 vs 収益見込み） |
| **Operational** | 運用・保守が現実的か |
| **Schedule** | 想定期間内に完了可能か |
| **Market** | 市場ニーズがあるか |

**テンプレートに含めるべき項目:**
- 各観点の評価結果（Go / Caution / No-Go）
- リスクと緩和策
- 総合判断と推奨

### poc-report（PoC結果報告）

**テンプレートに含めるべき項目:**
- PoCの目的・仮説
- 実施内容・環境
- 結果（成功/失敗、数値データ）
- 判明した制約・課題
- 判断（採用 / 不採用 / 追加検証）
- → 関連ADRへのリンク

## research から他ドキュメントへの接続

```
research（調査）
  ↓ 判断材料を提供
adr（意思決定）
  ↓ 技術選定・方針決定
spec（仕様書）
  ↓ 実装の根拠
実装
```

- `tech-evaluation` → 必ず対応するADRを作成
- `feasibility-study` → Go判断の場合、spec作成に進む
- `poc-report` → 結果に基づきADRで採用/不採用を決定
- `competitive-analysis` → steering/product.md の差別化ポイント更新に活用
